Figura gives Minecraft Java Edition players the ability to replace the default player model with a fully custom avatar created in [Blockbench](https://www.blockbench.net/). Other players running Figura can see your avatar automatically — no server-side installation required. An optional Lua scripting API lets you animate, react to events, and interact with the world in real time.

## How Figura works

Figura avatars are stored in your local `avatars/` folder and uploaded to the Figura backend server when you equip them. Other players running Figura automatically download and render your avatar. No configuration on any Minecraft server is required.

<Steps>
  <Step title="Install Figura">
    Download the Figura mod for your preferred mod loader (Fabric, Forge, Quilt, or NeoForge) and add it to your mods folder.
  </Step>
  <Step title="Create an avatar in Blockbench">
    Use the Blockbench modelling tool to design your custom player model. Export it into an avatar folder containing an `avatar.json` file.
  </Step>
  <Step title="Add Lua scripts (optional)">
    Write Lua scripts inside your avatar folder to animate parts, respond to game events, play sounds, and more.
  </Step>
  <Step title="Equip and upload">
    Open the Figura menu in-game, select your avatar, and upload it. Other Figura users will see it immediately.
  </Step>
</Steps>

<Note>
  Figura requires an online Mojang account. Non-premium (cracked) accounts and offline-mode servers cannot upload or download Figura avatars because Figura uses Mojang authentication to verify account ownership.
</Note>
