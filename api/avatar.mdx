---
title: "AvatarAPI — avatar metadata, storage, and budgets"
description: "The avatar global provides metadata about the running avatar, a key-value store for script state, and permission/budget introspection helpers."
---

The `avatar` global gives your script information about itself: its name, authors, file size, NBT data, and the permission level granted by the viewer. It also exposes a simple key-value store (`store`/`retrieve`) so scripts can share data between Lua files without polluting the global namespace.

---

## getNBT

Returns the avatar's NBT compound tag converted to a Lua table. The table mirrors the structure of the avatar's `.nbt` file.

<ResponseField name="return" type="table">
  The avatar NBT as a nested Lua table.
</ResponseField>

```lua
local nbt = avatar:getNBT()
print(nbt["name"])
```

---

## store

Stores a value under a string key in the avatar's internal storage table.

<ParamField path="body.key" type="string" required>
  The key to store the value under. Must not be nil.
</ParamField>

<ParamField path="body.value" type="any">
  The value to store. Pass `nil` to clear the key.
</ParamField>

<ResponseField name="return" type="AvatarAPI">
  Returns the `AvatarAPI` instance for chaining.
</ResponseField>

```lua
avatar:store("health", player:getHealth())
```

## setColor

Sets the avatar badge color. Accepts either a `Vector3` of RGB components (0–1 range) or three separate `number` arguments. An optional `badgeName` string targets a specific badge slot instead of the default color.

<ParamField path="body.color" type="Vector3 | number">
  Either a `Vector3` with RGB values, or the red component as a number (0–1).
</ParamField>

<ParamField path="body.g" type="number">
  Green component (0–1). Only used when passing individual numbers.
</ParamField>

<ParamField path="body.b" type="number">
  Blue component (0–1). Only used when passing individual numbers.
</ParamField>

<ParamField path="body.badgeName" type="string">
  Optional badge slot name. When provided, sets the color for that specific badge rather than the default avatar color.
</ParamField>

<ResponseField name="return" type="AvatarAPI">
  Returns the `AvatarAPI` instance for chaining.
</ResponseField>

```lua
-- Set default badge color to red
avatar:setColor(1, 0, 0)

-- Set color using a Vector3
avatar:setColor(vec(0.5, 1, 0.5))

-- Target a specific badge slot
avatar:setColor(vec(0, 0.8, 1), "pride")
```
