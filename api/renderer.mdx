---
title: "RendererAPI — camera, HUD, and render flags"
description: "The renderer global controls how the avatar and scene are drawn: fire, vehicle, HUD, camera transforms, FOV, shadow radius, and arm visibility."
---

The `renderer` global lets your script control a wide range of visual behaviors: whether fire, vehicle, crosshair, and HUD elements are drawn; the camera's position, rotation, pivot, and matrix; field of view; shadow radius; post-processing effects; arm and arrow visibility; and custom fire textures. Most setters return the `RendererAPI` instance so calls can be chained.

---

## Boolean render flags

The following public fields can be read and set directly as properties, or through their corresponding getter/setter methods.

| Field | Default | Description |
|---|---|---|
| `renderFire` | `true` | Whether the fire overlay renders when the player is on fire. |
| `renderVehicle` | `true` | Whether the vehicle/mount model renders. |
| `renderCrosshair` | `true` | Whether the crosshair renders. |
| `forcePaperdoll` | `false` | Forces the paperdoll preview to display. |
| `renderHUD` | `true` | Whether the HUD renders. |

```lua
renderer.renderFire = false
renderer.forcePaperdoll = true
```

## shouldRenderFire / setRenderFire

Gets or sets whether the fire overlay is rendered when the player is on fire.

<ParamField path="body.renderFire" type="boolean">
  `true` to show fire overlay, `false` to suppress it.
</ParamField>

<ResponseField name="return (getter)" type="boolean">
  Current fire render state.
</ResponseField>

<ResponseField name="return (setter)" type="RendererAPI">
  Returns `RendererAPI` for chaining.
</ResponseField>

```lua
renderer:setRenderFire(false)
print(renderer:shouldRenderFire()) -- false
```

## getShadowRadius / setShadowRadius

Gets or sets the radius of the shadow rendered beneath the avatar. Clamped to the range 0–12. Pass `nil` to reset to the default.

<ParamField path="body.radius" type="number | nil">
  Shadow radius in blocks (0–12). `nil` resets to default.
</ParamField>

<ResponseField name="return (getter)" type="number">
  Current shadow radius (defaults to `0.5` if not set).
</ResponseField>

<ResponseField name="return (setter)" type="RendererAPI">
  Returns `RendererAPI` for chaining.
</ResponseField>

```lua
renderer:setShadowRadius(2.0)
print(renderer:getShadowRadius()) -- 2.0
```

## isFirstPerson

Returns whether the local player is currently in first-person view and is the camera entity.

<ResponseField name="return" type="boolean">
  `true` if the avatar's owner is the current camera entity and in first-person mode.
</ResponseField>

```lua
if renderer:isFirstPerson() then
  -- hide head in first person
  models.Head:setVisible(false)
end
```

## isCameraBackwards

Returns whether the camera is currently in the mirrored/backwards third-person view (toggled with F5).

<ResponseField name="return" type="boolean">
  `true` if the camera is in mirrored view.
</ResponseField>

```lua

  print("Looking at your own face")

```

## getCameraPos / setCameraPos

Gets or sets an override for the camera position offset. Accepts a `Vector3` or three separate numbers. Pass `nil` (or no arguments) to clear the override.

<ParamField path="body.pos" type="Vector3">
  Camera position as a `Vector3`. Alternatively, pass `x`, `y`, `z` as separate numbers.
</ParamField>

<ResponseField name="return (getter)" type="Vector3 | nil">
  Current camera position override, or `nil` if not set.
</ResponseField>

<ResponseField name="return (setter)" type="RendererAPI">
  Returns `RendererAPI` for chaining.
</ResponseField>

```lua
renderer:setCameraPos(0, 1, 0)
-- or
renderer:setCameraPos(vec(0, 1, 0))
-- clear override
renderer:setCameraPos(nil)
```

## setPostEffect

Applies a fullscreen post-processing shader effect by name. The effect must be a valid Minecraft shader located in `shaders/post/`. Pass `nil` to remove the current effect.

<ParamField path="body.effect" type="string | nil">
  Shader name without path or extension, e.g. `"blur"`, `"creeper"`. Pass `nil` to clear.
</ParamField>

<ResponseField name="return" type="RendererAPI">
  Returns `RendererAPI` for chaining.
</ResponseField>

```lua
renderer:setPostEffect("blur")
-- later, remove it
renderer:setPostEffect(nil)
```

<Warning>
  Throws a Lua error if the specified shader does not exist in the active resource pack.
</Warning>

## getRootRotationAllowed / setRootRotationAllowed

Gets or sets whether the avatar's root model group is allowed to rotate with the player's body. Defaults to `true`.

<ParamField path="body.bool" type="boolean | nil">
  `false` to lock the root model against body rotation. `nil` resets to the default (`true`).
</ParamField>

<ResponseField name="return (getter)" type="boolean">
  `true` if root rotation is currently allowed.
</ResponseField>

<ResponseField name="return (setter)" type="RendererAPI">
  Returns `RendererAPI` for chaining.
</ResponseField>

```lua
renderer:setRootRotationAllowed(false)
print(renderer:getRootRotationAllowed()) -- false
```
